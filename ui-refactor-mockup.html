Pantry — UI refactor mockup

This is an interactive mock of the proposed UI. Tap the buttons above the phone, the bottom nav, or the orange + FAB to move between screens. On the Pantry screen, drag an item row left or right to preview swipe actions. The Toast button fires the undo snackbar that would appear after any destructive action.

What's new

4-tab nav (Today · Pantry · Plan · Shop) Global + FAB → Quick Add (Scan · Type · Voice) Swipe-to-act on every list Inline +/− steppers, no sheet to change qty Pantry grouped by location, Shop grouped by aisle Copy-last-week, Make-shopping-list in one tap Undo snackbar replaces confirm() dialogs Smart defaults on scan (last location, product-type expiry)

What's gone

Scan as a dedicated bottom-nav tab Three-icon cluster on pantry rows <datalist> autocomplete Multi-step recipe import wizard confirm() dialogs Shopping list hidden inside Plan

How to read this

This mockup is paired with ui-refactor-plan.docx, which has the full audit, competitive review, design principles, screen specs, interaction catalog, phased implementation plan, risks, and success metrics. The visuals here are illustrative — the doc is the source of truth for the plan.
